# v1.2.0 (2026-06-02)

## New Features

- `alcom redo` コマンドを追加: `undo` で取り消したスナップショットを1つ復元 (#15)
- `alcom save --auto` オプションを追加: 変更ファイルの差分情報からコミットメッセージを自動生成（変更量が多い順に `path (+added/-deleted)` 形式でソート） (#18)
- `alcom status --short` オプションを追加: ファイル件数とtruncated listで簡易表示 (#21)
- `alcom log --long` オプションを追加: メッセージを省略せずに全文表示 (#21)
- `CODING_AGENT_ROOT` 環境変数サポート: 設定されている場合、そのディレクトリの `.git` を作業ディレクトリとして使用 (#21)
- `alcom status` のデフォルト比較先を前回スナップショットに変更し、`--depth` / `--base` オプションを追加 (#23)

## Bug Fixes

- untrackedファイルのdiff統計で実際の行数をカウントするよう修正（空ファイルを行数1と誤カウントする問題を解消） (#20)
- PreToolUse hooksをcheckoutブロックとswitchガードで分離し、チェーンコマンドでregexがバイパスされる問題を修正 (#22)
- redoコマンドの操作順序を修正（resetHard成功後にpopUndoStackを実行） (#15)

## Changes

- undoコマンドの出力に取り消したスナップショットのメッセージ、ファイル差分情報、redoによる復元案内を追加 (#15)
- undo/redo実行後にstderrで残りスナップショット数と現在の変更状況を表示 (#21)
- logコマンドの表示を降順（新しい順）に変更し、UTCからローカル時刻に変更。メッセージ省略制限を30文字から60文字に拡大、`--alcom--` プレフィクスを非表示 (#21)
- checkout hookメッセージをfinish優先からundo優先フローに変更（`alcom log` 確認 → 不要分だけundo → 残りをfinishの流れを案内） (#15)
